第1章　jQuery

なんで jQuery？

  • 簡単な記述でDOMの選択・操作が可能
  • ブラウザ間の差異を吸収してくれる
    ＝ クロスブラウザ対応を意識せずに書ける
  • プラグインが豊富
    ＝ 自分でコードを書かなくてもいろんな処理ができる
  • チェーンメソッドでサクサク書ける
第1章　jQuery

jQueryの弱点

  • アニメーションが遅い
  • セレクタの書き方によっては動作が遅くなる
  • そもそもjQuery自体を読み込む分だけ動作が遅くなる
  • プラグインが古くなってきてる・メンテナンスされてないのが多い（非効率な書き方をされてるかもしれない）
  • AngularJSなどのフレームワークと相性が悪いことがある

第1章　jQuery

jQuery 2系→3系の変更点

  • .load(), .unload(), .error()の削除（.on()を利用しましょう）
  • .on('ready', fn)の削除（$(fn)を利用しましょう）
  • .show(), .hide()の仕様変更（css + .addClass()などで対応しましょう）

などなど、色々ありますが割愛。知りたい人は調べてみてね。

[jQuery Core 3.0 Upgrade Guide | jQuery] https://jquery.com/upgrade-guide/3.0/

Vanilla JS って何？

  • 強力なJavaScript製アプリケーションのための、高速・軽量・クロスプラットフォームなフレームワーク

Vanilla JS めちゃ軽い

第2章　次の一歩

なんで Vanilla JS？

  • jQueryの弱点である「遅い」を解決できる
  • 古いブラウザをサポートする必要がなくなってきた
    ＝ jQueryを使う必要がないかもしれない
  • JavaScriptも進化している。新機能を使えるのなら使っていったほうがいいよね
第2章　次の一歩

Vanilla JSの弱点

  • とはいえ、まだブラウザ間での非互換性はあったりする
    ＝ jQuery等のツールがないとツライ場面もまだまだある
  • 慣れるまでは、ブラウザごとの実装状況を確認したり、挙動を確認したりする手間が必要かもしれない
  • がんばって書くのは時間がかかるかもしれない
第2章　次の一歩

それでも Vanilla JS 推し？

  • あくまで私の個人的な意見ですが…
  • 脱jQueryをする必要はないと思ってる
  • Vanilla JSとjQueryをうまく使い分けできたらいいんじゃないかなって
  • 必要なさそうなら無駄にjQueryを使わなくてもいいし

// IE10+用
function closest(el, selector) {
  const matchesSelector = el.matches || el.webkitMatchesSelector || el.mozMatchesSelector || el.msMatchesSelector;

  while (el) {
    if (matchesSelector.call(el, selector)) {
      return el;
    } else {
      el = el.parentElement;
    }
  }
  return null;
}

第4章　ECMAScript

第4章　ECMAScript

ECMAScript（ES）って何？

  • ECMAScriptは、ECMA Internationalのもとで標準化されているJavaScriptの言語仕様
  • JavaScriptはECMAScriptの仕様に基づいた言語のひとつ（JavaScript ≠ ECMAScript）
  • 現在多くのブラウザで安定して利用できるバージョンは2009年策定のECMAScript 5th Edition（ES5）
第4章　ECMAScript

ECMAScript 2015 / 2016 / next

  • ECMAScript 6th Edition（ES6）以降は、毎年アップデートしていくことになったため、ECMAScript 2015という呼称に変更（ES6 = ES2015）
  • 2015年リリースの仕様がECMAScript 2015（ES2015）、2016年リリースの仕様がECMAScript 2016（ES2016）
  • 次期ECMAScriptを「ECMAScript next」とか「ES.next」と言ったりもする
第4章　ECMAScript

ECMAScript 2015で追加された仕様 (1)

  • let, constによる変数宣言

    • 従来の変数宣言varに代わるもの。と思ってOK。
    • letは再宣言不可、定数を示すconstは再代入も不可。
    • {}によるブロックスコープが有効に（サヨナラ即時関数）。
第4章　ECMAScript

ECMAScript 2015で追加された仕様 (2)

  • Arrow Function

    • 今までfunction() {...}と書いていたものが
      () => {...}と書ける。
    • これで作る関数はthisコンテキストを切り替えない。
      （サヨナラvar self = this;）
    • 引数が1つの場合は()を省略可能。param => {...}でもOK。
第4章　ECMAScript

ECMAScript 2015で追加された仕様 (3)

  • 分割代入

    複数の変数への代入を１つにまとめることができる。 
    var a = 1, b = 2, c = 3
//  ↓
const [a, b, c] = [1, 2, 3]
第4章　ECMAScript

ECMAScript 2015で追加された仕様 (4)

  • Spread operator / スプレッド演算子

    関数の呼び出し、配列への代入、分割代入に対して、
    ...により、配列が展開される。 
    var args = [0, 1, 2];
myFunction.apply(null, args);
//  ↓
const args = [0, 1, 2];
myFunction(...args);
第4章　ECMAScript

ECMAScript 2015で追加された仕様 (5)

  • 可変長引数

    可変長の引数を受け取ることができるようになった。 
    // こんな感じに
const func = (x, ...y) => {
  console.log(x);  // 3
  console.log(y);  // ['hello', true]
  return x * y.length;
}
func(3, 'hello', true);  // 6
第4章　ECMAScript

ECMAScript 2015で追加された仕様 (6)

  • デフォルト引数

    関数に指定する引数の初期値を設定できるようになった。 
    // こんな感じに
const func = (x, y = 12) => {
  return x + y;
}
func(3, 5);  // 8
func(3);     // 15
第4章　ECMAScript

ECMAScript 2015で追加された仕様 (7)

  • Template literal

    文字列をバッククオート` `で囲むと、
    • 複数行の文字列でも、改行がそのまま書ける。
    • ${}内の変数や式を展開してくれる。
    • 引用符を含むHTMLコードが書きやすい。
    // こんな感じに
const list = `<ul class="list">
  <li class="list-item">${listName}-01</li>
  <li class="list-item">${listName}-02</li>
  <li class="list-item">${listName}-03</li>
</ul>`
第4章　ECMAScript

ECMAScript 2015で追加された仕様 (8)

  • class構文＋extends（クラスの継承）
  • Promise

などなど、たくさんの新しい文法が追加されている。
